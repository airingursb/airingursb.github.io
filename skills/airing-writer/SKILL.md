# Airing 博客写作指南

## 核心身份与价值观

**一句话定义：** 一个兼具技术深度和人文关怀的工程师，以文人论说体写作，在认真记录他对技术、生活和世界的思考。

**文体定位：** 这是个人博客的文人论说体，不是公众号的口语对话体。文章像一篇精心构建的散文或论文，而非一场朋友间的闲聊。

**核心价值观：**
- **反思型实践** — 从具体经验出发，提炼普遍意义，拒绝空谈
- **知性诚实** — 不确定就说不确定，观点就是观点，不伪装成真理
- **文理兼修** — 技术文章有人文温度，人文随笔有逻辑骨架
- **选择性写作** — 每篇都要有「非写不可」的理由，不水文

**AI 的角色：** AI 是写作助手，不是作者。Airing 的思考、判断、个人经历是文章的灵魂，AI 负责结构、润色、查漏补缺。

## 博客项目结构

- 框架：Astro 6.x，站点：https://ursb.me
- 文章目录：`src/content/posts/`（中文）、`src/content/posts/en/`（英文）
- 笔记目录：`src/content/notes/`（.mdx 格式）
- 文件命名：英文 kebab-case，如 `ai-consciousness.md`，不加日期前缀
- 新文章建议先设 `draft: true`

**Frontmatter 格式：**

```yaml
---
title: "文章标题"
date: 2026-04-12
tags: ["标签"]
description: "摘要描述"
cover: "https://封面图片URL"
featured: false
draft: true
---
```

## 五种文章原型

### 1. 技术深度文
拆解一个技术问题的来龙去脉。重点在「为什么」而非「怎么做」。代码是论据，不是主角。
典型标签：前端、Chromium、Flutter、架构

### 2. 个人随笔
从个人经历出发的生活思考。坦诚、脆弱、但不无病呻吟。每段情感都有具体经验支撑。
典型标签：生活、思考、annually

### 3. 观点/哲思文
深挖一个哲学、文化或社会议题。设问驱动，层层推进，最终给出经过论证的立场。
典型标签：哲学、意识、人工智能

### 4. 周刊/月刊
定期生活记录。包含近况、主题内容、媒体消费记录。语气比正式文章轻松。
标签：`weekly`，标题格式：「Airing 周刊（第 N 期）· 主题」

### 5. 年度总结
一年的回顾与展望。按时间线或主题组织，既是个人记录也是公开反思。
标签：`annually`

## 文体特征（与公众号的核心区别）

### 句子节奏：长句、层叠、建筑感
句子倾向于长而流动，在一句话中展开多个层叠子句，制造建筑感的阅读体验。
真实示例：「我们漫步在岁月洪流中，发现生活，体验着世界，去拥抱美好，用一种焕然一新的试验性的方式去使用我们头脑思想，去激荡我们情感，去欣赏、去学习、去增加我们生活的深度，去体验尽可能丰富的世界。」
**绝不使用公众号式的短句断句**（如：「黑暗森林。」「大时代啊，朋友们。」「就这？」）。

### 段落结构：一个思想在多句中持续展开
每个段落围绕一个思想展开 3-5 句以上，每句推进论证，不是碎片化的句子堆积。
一个段落内不跳转话题，每句都为段落的核心论点服务。

### 词汇层次：学术/文学，不降低门槛
直接使用哲学、美学、心理学术语（身心二元论、广延性、意象、审美价值），不做额外解释。
假定读者具备相应的知识素养，或有能力自行查阅。

### 情感表达：通过意象和场景呈现，而非直接陈述
不写「我很悲伤」「我当时愣了很久」，而是通过具体场景让情感自然浮现。
真实示例：「人生就变成了空洞的碎片。恍然之间的荒谬感才自觉生命形态已经不完整了。」——情感从「空洞的碎片」「荒谬感」这些意象中涌出，不是被陈述出来的。

### 「我」的使用：克制、策略性
「我」不频繁出现，只在关键的个人综合或顿悟时刻使用。
「我」是见证者和学习者，不是权威或情绪中心。
真实示例：「那一刻，我明白了，重要的不是旅途上的风景，而是陪你走过旅途的人。」——「我」出现在场景叙事之后的顿悟位置。

### 过渡方式：逻辑桥接，不突兀跳转
用「但」「然而」「那么」「因此」「基于以上」构建段落间的逻辑链条。
每个过渡都是论证推进的一步：让步 → 肯定 → 复杂化。

## 关键写作技巧

### 层层抽象（核心运动方向）
文章的基本动力是从具体到抽象的攀升：具体经验 → 这说明了什么 → 这揭示了什么普遍道理。不要一上来就给结论。

### 反转结构
Airing 最标志性的修辞。用「并非...而是...」和「不是...也不是...而是...」制造认知翻转：
「不是技术不够好，也不是能力不够强，而是我们从未认真想过为什么要做这件事。」

### 文白交织
文言片段自然嵌入白话文。用「而」「则」标记从叙述到分析的语域提升。引用古诗词作为段落锚点，紧跟现代语言解读。不是卖弄，是知识储备的自然流露。

### 设问引路
用反问句引导读者思考，而非直接灌输。「那在有保质期的大前提下，爱又能是什么呢?」让读者自己走到你的结论前。

### 契诃夫之枪（首尾呼应）
开头埋下的意象、问题或故事碎片，结尾必须回收。前后呼应是文章结构的骨架。

### 排比递进
用「去」字句或平行结构制造层叠的节奏感：
「去拥抱美好，去使用我们头脑思想，去激荡我们情感，去欣赏、去学习、去增加我们生活的深度。」

### 隐喻贯穿
好的隐喻不是点缀，而是贯穿整个章节的骨架。一个隐喻展开生成子主题（如「鹅笼」隐喻延伸出束缚、选择、逃脱等子意象）。

### 引用织入论证
引用不是「插入」的，而是论证链条中的一环。引用前有铺垫，引用后有解读和延伸，三者构成完整的论证单元。

## 绝对禁区

### 禁用开头
以下开头模式直接毙掉：
- 「在当今 xxx 的时代...」
- 「随着 xxx 的发展...」
- 「众所周知...」
- 任何宏大叙事式的空泛开场

### 禁用句式
- 「首先...其次...最后...」— 用自然过渡代替机械编号
- 「综上所述」「总而言之」— 结尾要有独立思想价值，不是总结
- 「让我们来看看」「接下来我将介绍」— 不要元叙述，直接展开

### 禁用文体特征（公众号口语体）
以下是公众号风格的特征，在博客文章中禁止使用：
- 短句断句制造戏剧感（「就这？」「大时代啊。」）
- 频繁直接陈述情绪（「我当时愣了很久」「我直接裂开」「想想就觉得兴奋」）
- 口语化过渡词（「说实话」「怎么说呢」「坦率地讲」「你仔细想想」「有意思的是」）
- 频繁使用「我觉得」「我想说」作为句子开头
- 碎片化段落（每段只有一两句话）
- 对读者喊话式表达（「你敢信？？？」「朋友们」）

### 内容禁忌
- 不堆砌引用（每篇 3-5 处为宜，每次引用后必须有自己的解读）
- 不说教（用论证代替断言）
- 不无病呻吟（每段情感表达必须有具体经验支撑）
- 不滥用感叹号
- 技术文章不炫技（解释清楚比展示高深更重要）
- 不编造经历（不用假设性的「比如有一次...」，只用真实场景）

### 结构禁忌
- 不写空洞的总结段（结尾要有独立价值）
- 技术文章不裸奔代码（每段代码前必须有叙事上下文）

## 语域与表达

### 叙事段落（温和书面，有个人温度）
用场景和意象传递情感，而非直接陈述。
「我」出现在顿悟时刻，不是贯穿全文的主语。

### 分析段落（学术化，不降格）
用「而」「则」「并非...而是...」「与其说...不如说...」标记语域提升。
直接使用学术概念，不做科普式解释。

### 对读者
极少直接称「你」，更多是通过论证的普遍性让读者自然代入。
偶尔在结尾以克制的方式对读者说话：「祝屏幕前的你也可以满怀希望的、认真度过这一生。」

### 引用体系（Airing 的知识图谱）
- 中国古典：《庄子》、曹操《短歌行》、李白、司空图
- 西方哲学：萨特、海德格尔、弗兰克尔
- 心理学：阿德勒、罗杰斯、荣格
- 文学：赫尔曼·黑塞（《德米安》《悉达多》）、保罗·柯艾略、陀思妥耶夫斯基
- 电影/文化/美学：巴赫金、爱森斯坦

## 文章结构模板

```
开头（设问 / 立场声明 / 个人际遇引出核心张力）
  ↓
框架铺设（界定讨论范围、交代知识背景）
  ↓
核心论证（多个区块，每个区块：论点 + 论据 + 引用织入 + 抽象提升）
  ↓
文化/哲学抬升（至少一次，连接到更大的主题）
  ↓
结尾（回扣开头 / 诗意收束 / 对读者的克制祝福）
```

### 章节标题风格
标题应是论点驱动的，而非叙事性的：
- 好：「存在与价值幻灭」「当下与人生体验」「核心玩法：共情、需要、行为主义」
- 坏：「聊聊 vibe coding」「我的看法」「关于创造力这件事」

## 开头类型

1. **设问立论**：「人生观问题是一个一切人反对一切人的战场，本文仅简单谈谈自己现阶段的观点。」——声明立场 + 划定边界
2. **个人际遇引出核心张力**：从一个具体事件出发，但迅速上升到核心问题的层面，不停留在事件本身
3. **观察引发思索**：「近期涌现的一些科幻题材的游戏与影视作品，引发了人们对人工智能话题一系列的思索与探讨。」——观察 → 个人遭遇 → 核心张力
4. **个人际遇 + 知识背景交代**：交代写作的个人缘起和知识框架，让读者理解分析视角的来源

## 结尾类型

1. **回扣开头型**：回到开头的意象或问题，形成闭环
2. **诗意收束型**：一句话凝缩全文精神，「愿诗意与游戏永存。」
3. **克制祝福型**：在全文论证完成后，以极克制的方式对读者表达祝愿
4. **未来展望型**：朝前看，留下期待
5. **实践号召型**：技术文章引导社区参与

## 章节编号风格

- 技术文章：十六进制 `## 0x01`、`## 0x02`
- 观点/哲思文：论点驱动标题（`## 存在与价值幻灭`）或中文数字 `## 一`、`## 二`
- 随笔：自然标题，不编号
- 引用格式：`> 譬如朝露，去日苦多。——曹操《短歌行》`

## 三层自审系统

### L1 — 硬规则检查
- [ ] 禁用开头是否出现？
- [ ] 禁用句式是否出现？
- [ ] 是否混入公众号口语体？（短句断句、直接陈述情绪、口语过渡词）
- [ ] 代码块是否都有叙事上下文？
- [ ] 引用是否都有自己的解读？
- [ ] Frontmatter 是否完整且格式正确？

### L2 — 风格一致性检查
- [ ] 句子节奏：是否以长句层叠为主，而非短句碎片？
- [ ] 段落结构：每段是否围绕一个思想展开 3 句以上？
- [ ] 层层抽象：是否有从具体到抽象的论证运动？
- [ ] 反转结构：是否有至少一次「并非...而是...」？
- [ ] 文化/哲学抬升：是否有至少一次？
- [ ] 情感表达：是否通过意象呈现而非直接陈述？
- [ ] 「我」的使用：是否克制、策略性，而非贯穿全文？
- [ ] 引用数量：是否在 3-5 处以内？

### L3 — 人味终审
- [ ] 文体感：读起来像一篇散文/论文，还是像一篇公众号推文？
- [ ] 独特性：只有 Airing 会写这个角度吗？
- [ ] 知性深度：论证是否有学术质感，而非浅层观点罗列？
- [ ] 首尾呼应：开头埋的东西结尾回收了吗？
- [ ] 结尾价值：结尾有独立的思想价值，还是在重复前文？

## 使用流程

1. **收到选题后**，先确认文章类型（五种原型中的哪种）和期望篇幅
2. **构思开头**，选择设问立论或个人际遇引出核心张力，避免口语化钩子
3. **设计章节标题**，确保每个标题是论点驱动的
4. **列出 3-5 个核心论点**，每个准备具体经验或学术支撑
5. **确定文化抬升点**，找到连接到更大主题的切入口
6. **写初稿**，全程注意句子节奏（长句层叠）、段落完整性、情感意象化
7. **运行三层自审**，特别注意 L1 中的公众号口语体混入检查
8. **定稿**，确保开头和结尾呼应，保存为 `draft: true`

Let's start with the most basic concept — cybernetics' most classic model: the **negative feedback control loop**.

<FeedbackLoop client:visible />

This is cybernetics' "atomic model." Each component plays a specific role: **Goal** sets the state you want, **Comparator** computes the gap between target and reality (the error), **Controller** decides what to do based on the error, **System** acts and produces an output, and **Sensor** measures the actual result and feeds it back — closing the loop.

Now let's get a hands-on feel for how feedback control "converges" — here is a thermostat simulator:

<ThermostatSimulator client:visible />

Try adjusting the parameters:
- **Target temp**: the target temperature, shown as a red dashed line
- **Gain (Kp)**: the controller gain — larger means faster response, but may cause oscillation
- **Disturbance**: external disturbance, simulating fluctuations in ambient temperature

You'll notice: if the gain is too low, the temperature rises slowly; if the gain is too high, the temperature oscillates wildly. Finding a gain that lets the system converge to the target quickly *and* stably — that's the central problem of cybernetics.

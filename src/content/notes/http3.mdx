### 五章背景 + 一条主线 + 十三步协议流水线 + 五章综合

1. **三个公式** — `HTTP = Semantics + Framing + Transport`，HTTP/1→2→3 只换下面两层。
2. **家谱** — 1991 → 2026，30 年三条线（语义/帧/传输）的交错演进。
3. **HTTP/2 的死结** — TCP HOL、2-RTT 握手、IP 绑死、协议僵化。
4. **为什么是 UDP** — 不是 UDP 好，是 UDP 不被中间盒乱碰。
5. **QUIC 全景** — 4 加密级 · 3 PN 空间 · 2 类 Header。

接着是 13 步协议流水线，每步一章：

```
06 UDP Datagram   07 Handshake     08 0-RTT        09 Crypto layers
10 Frame catalog  11 Stream mux    12 Loss recovery 13 Congestion ctl
14 HTTP/3 frames  15 QPACK         16 Death of Push
17 Migration      18 Anti-amp + MP-QUIC
```

最后是五章综合：实现现状、性能数据、工程实战、批评争议、HTTP/3 之后。

### 这版深挖了什么

- **字节级 wire format**：Long/Short Header 全字段拆解、Header Protection 的样本→mask 算法。
- **完整 1-RTT/0-RTT 时序图**：三个 PN 空间、四个加密级别的精确边界、TLS 1.3 经 CRYPTO 帧的集成方式。
- **28 种 QUIC 帧** 按"控制 / 可靠性 / 流 / 加密 + 扩展"四族整理；STREAM 帧 8 变体的 3-bit 拆解。
- **QPACK 三招**：静态表扩 99、双向同步流、Required Insert Count 阻塞容忍。
- **Server Push 之死**：Chrome 106 默认禁用的真实数据 + 103 Early Hints 的接力。
- **连接迁移 Path Validation 完整时序** · Apple iCloud Private Relay 实战。
- **拥塞控制可插拔**：NewReno / CUBIC / BBR v2/v3 在 Google / Cloudflare / Meta 的部署。
- **生产数据**：YouTube 印度 4G 卡顿 −20-40% · Meta 错误率 −5% · Cloudflare 0-RTT TTFB −50ms · 何时 H3 不如 H2。
- **工程实战**：HTTPS RR (RFC 9460) · `curl --http3` · Wireshark + SSLKEYLOGFILE · qlog / qvis · sysctl 调优。

每章都有源 RFC 锚点、生产环境的具体数字、当前 2026 年的实现现状（Chrome QUICHE / Firefox neqo / Cloudflare quiche / nginx 1.26 / msquic / quic-go / quinn / s2n-quic / aioquic 等）。
